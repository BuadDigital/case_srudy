# سياق المشروع

نظام إدارة معاملات تقييم عقاري (عربي RTL) — الملف الرئيسي: `Case Study.html`، والتوثيق: `دورة اسناد المهام.md`.

## ما أُنجز
- **إدارة المفاتيح والأظرف**: تسجيل ظرف (مرجع ENV-2026-xxx) مع بحث حي برقم الطلب، سلسلة عهدة وسجل زمني على مستوى المفتاح والظرف، تسليم داخلي (تأكيد استلام) وخارجي (إثبات تسليم إلزامي: اسم، جهة، صفة، جوال)، مؤشرات مراقبة، صفوف باهتة للمسلَّم.
- **المهام**: 3 أنواع (زيارة محكمة، استفسار، مهمة). حالات: قيد التنفيذ (فورية بلا موافقة) / متوقفة (سبب إلزامي، حد يوم عمل) / منجزة / ملغاة.
- **الضوابط**: تأكيد استلام من المنفّذ؛ الإغلاق «منجزة» للمنفّذ والإلغاء للمنشئ (نافذتان منفصلتان)؛ التذكير للمنفّذ والمنشئ؛ مسؤولية التنفيذ عند إعادة التوجيه تُسجَّل للأول وقابلة للتعديل عند الإغلاق.
- **إغلاق زيارة المحكمة**: نتائج متعددة الاختيار (استُلم ظرف ↔ عند طرف آخر متعارضان)، إفادة موحدة أو لكل صك، جهات اتصال (اسم/صفة/جوال/ملاحظات)، وعند استلام ظرف يظهر زر «تسجيل الظرف الآن» يربط بتسجيل الظرف.
- **الأتعاب**: أتعاب الزيارة تُستحق بمهمة زيارة المحكمة لمن نفّذها؛ تصوير الظرف يثبت أتعاب الشركة لدى مركز الإسناد والتصفية (تأكيدها لموظف المالية).

دورة المعاملة قصيرة: 4–5 أيام عمل — تُراعى في التذكيرات.
