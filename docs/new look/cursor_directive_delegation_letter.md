# توجيه اعتماد منطق خطاب التفويض الداخلي

نظام إجادة | وحدة المراجعة الحكومية
الغرض: مواءمة التنفيذ الحالي مع المنطق المعتمد بعد النقاش.
مرجع الوضع الحالي: `authorization_letter_current_state.md`

---

## 0. المنطق المعتمد (المرجع الحاكم)

هذه هي القواعد النهائية. أي تعارض مع التنفيذ الحالي يُحسم لصالح هذا القسم.

1. **الخطاب موجّه للدائرة.** مفتاح تجميع الخطاب = **المحكمة + الدائرة**، وليس المحكمة فقط.
2. **الأمر والدائرة خاصية لكل صك.** كل عقار (رقم صك) يحمل أمر عمله الخاص ودائرته الخاصة. لذلك الخطاب الواحد (لمحكمة + دائرة محددة) قد يجمع عقارات بأوامر عمل **متعددة**.
3. **أمر العمل عمود في الجدول** — يختلف من صف لآخر، لا يُوضع في الترويسة.
4. **المحكمة/الدائرة سطر مستقل في نص الخطاب**، وليست أعمدة في الجدول.
5. **جدول الخطاب:** `التسلسل | أمر العمل | رقم الصك | المالك | رقم الطلب`.
6. **القالب الرسمي S-0521** هو المرجع (ترويسة، جهة توجيه، نص، شركة، مفوَّض، ختم/توقيع).
7. **الرقم المرجعي** بصيغة الترميز الموحّد (انظر §5)، لا `S-05xx` القديم.

## 1. الفرق بين الحالي والمعتمد

| الجانب | الحالي | المعتمد | إجراء |
|---|---|---|---|
| مفتاح التجميع | محكمة فقط | محكمة + دائرة | **تغيير** |
| الدائرة | من أول عقار في المجموعة (عيّنة) | جزء من المفتاح | **تغيير** |
| أوامر العمل | أمر واحد لكل خطاب (مربوط بـ PO) | متعددة، حسب الصك | **تغيير** |
| أمر العمل في الجدول | غير موجود بالصف | عمود لكل صف | **إضافة** |
| القالب | HTML مبسّط | S-0521 رسمي | **استبدال** |
| المرجع | لا يوجد | ترميز موحّد §5 | **إضافة** |
| تاريخ هجري | ميلادي فقط | هجري + ميلادي | **إضافة** |
| بيانات المفوَّض | غير موجودة | من ملف المراجع | **إضافة** |
| بيانات الشركة | غير موجودة | ثابتة في القالب | **إضافة** |
| snapshot الصفوف | يُعاد بناؤها وقت الطباعة | تُثبَّت عند الإصدار | **تغيير** |
| ختم/توقيع/اعتماد | غير موجود | مطلوب قبل الإصدار | **إضافة** |

## 2. تغيير مفتاح التجميع (الأهم)

**الملف:** `apps/mfe-case-study/src/lib/prototype/internal-delegation-letters.ts`

الحالي يجمع حسب `court` فقط، ويأخذ `circuit` من أول عقار. المطلوب:

- مفتاح المجموعة = تركيبة **(court + circuit)** معاً.
- كل عقار يبقى مع مجموعته حسب دائرته الفعلية، لا حسب دائرة العيّنة.
- `id` الخطاب يصبح: `"{court}::{circuit}"` (بدون `poNumber`، لأن الخطاب لم يعد مرتبطاً بأمر عمل واحد — انظر §3).

```ts
type InternalDelegationLetter = {
  id: string;                 // "{court}::{circuit}"
  city: string;
  court: string;
  circuit: string;            // جزء من المفتاح، ليس عيّنة
  selectedProperties: {
    propertyId: string;
    workOrder: string;        // أمر عمل هذا الصك تحديداً
    deedNo: string;
    owner: string;
    requestNo: string;
  }[];
  reference?: string;         // يُثبَّت عند الإصدار (§5)
  createdAt: string;
};
```

> **جوهر التغيير:** أمر العمل ينتقل من مستوى الخطاب إلى مستوى العقار داخل الخطاب.

## 3. فكّ الارتباط بأمر عمل واحد

**الملفات:** التخزين والـ API.

الحالي: سجل واحد لكل PO (`InternalDelegationLetterSets` مفهرس بـ `PoNumber`)، والخطابات JSON داخله.

المطلوب: الخطاب يعبر حدود أمر العمل الواحد. الخيارات للمطور:

- **المفضّل:** ربط سجل الخطاب بـ **(المحكمة + الدائرة)** ضمن نطاق المراجعة/المندوب، لا بـ PO.
- تحديث الفهرس والـ query وفقاً لذلك.
- `GET` يقبل مفتاح المراجعة/المندوب بدل `poNumber` فقط.
- `PUT` يستبدل خطابات هذا النطاق.

الصلاحية تبقى `ManageWorkOrders` (أو ما يناظرها للمراجعة الحكومية).

## 4. قالب الطباعة الرسمي

**الملف:** `apps/mfe-case-study/src/lib/prototype/internal-delegation-letter-html.ts`

استبدال القالب المبسّط الحالي بقالب S-0521. المرجع البصري الجاهز: `authorization_letter.html`.

**بنية القالب:**
- ترويسة: شعار إجادة + رقم المرجع + التاريخ (هجري + ميلادي).
- جهة التوجيه: `فضيلة رئيس محكمة التنفيذ — {city}    المحترم`.
- سطر مستقل: `المحكمة / الدائرة: {court} / {circuit}`.
- التحية + الموضوع (تفويض).
- نص التفويض بمتغيرات المفوَّض والشركة.
- الجدول: `التسلسل | أمر العمل | رقم الصك | المالك | رقم الطلب`.
- ختم الشركة + التوقيع.
- تذييل بيانات التواصل.

**نص التفويض:**
> بالإشارة إلى الموضوع أعلاه وبناءً على التفويض الصادر من مركز الإسناد والتصفية (إنفاذ) المذكورة أدناه، نفوض نحن {company_name} سجل تجاري رقم {company_cr} السيد {agent_name} {agent_nationality} الجنسية، ويحمل الهوية الوطنية رقم {agent_id} ورقم جوال {agent_mobile} لاستلام مفاتيح العقارات بمحافظة {city} والمحافظات التابعة لها.

**مصادر المتغيرات:**
- `agent_*` من ملف المراجع الحكومي (المستخدم).
- `company_name` = شركة إجادة المهنية للتقييم، `company_cr` = 4030297680 (ثابت).
- `court / circuit / city` من مفتاح الخطاب.
- صفوف الجدول من `selectedProperties` (كل صف: أمر عمله + صكه + مالكه + طلبه).

## 5. الرقم المرجعي

اعتماد الترميز الموحّد بدل `S-05xx`. المرجع الكامل: `document_reference_spec.md`.

```
[الإدارة] - [النوع] - YYMMDD - NNN
```

- خطاب المراجعة الحكومية: الإدارة `DS` (دراسة الحالة) أو حسب الإدارة المُصدِرة، النوع `LT`.
  - مثال: `DS-LT-260719-001`
- التاريخ ميلادي (YYMMDD)، العدّاد يتصفّر يومياً، ثلاث خانات (001–999).
- العدّاد مستقل لكل تركيبة (إدارة + نوع)، وتوليده **ذرّي** (atomic) لمنع التكرار.
- الرقم يُثبَّت في سجل الخطاب **لحظة الإصدار** ولا يتغير.

الأرقام القديمة `S-05xx` تبقى كما هي في الأرشيف دون تحويل.

---

## 6. Snapshot عند الإصدار

عند إصدار الخطاب (لا وقت الطباعة):
- تُثبَّت صفوف الجدول (صك + مالك + طلب + أمر عمل) كنسخة ثابتة في سجل الخطاب.
- يُثبَّت الرقم المرجعي والتاريخان (هجري/ميلادي) واسم المفوَّض.
- تعديل لاحق في دليل المحاكم أو بيانات العقار **لا يغيّر** الخطاب المُصدَر.

---

## 7. الترتيب المقترح للتنفيذ

1. تغيير مفتاح التجميع إلى (محكمة + دائرة) — §2.
2. نقل أمر العمل لمستوى العقار داخل الخطاب — §2.
3. فكّ ارتباط التخزين/الـ API بـ PO واحد — §3.
4. تطبيق القالب الرسمي S-0521 — §4.
5. إضافة الرقم المرجعي بالترميز الموحّد — §5.
6. إضافة snapshot الإصدار + التاريخ الهجري — §6.
7. (لاحقاً) الختم/التوقيع/الاعتماد قبل الإصدار.

## 8. ملاحظات

- **جهة التوجيه** تكرر اسم المحكمة الوارد في سطر المحكمة/الدائرة — مقبول تصميمياً (الأولى مخاطبة، الثاني تحديد). يمكن تبسيطها إلى "فضيلة رئيس الدائرة" إن رُغب لاحقاً.
- **الختم والتوقيع** يُدرجان من أصول المنشأة (`ejadah-stamp.png` / `ejadah-signature.png`).
- الاعتماد قبل الإصدار (خطوة 7) اختياري في المرحلة الحالية، لكن snapshot والرقم المرجعي غير قابلين للتأجيل — هما أساس تكامل الأرشيف.
