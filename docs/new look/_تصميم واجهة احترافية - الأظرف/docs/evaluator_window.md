# نافذة المقيم العقاري — توثيق المطور

## نظرة عامة

نافذة مخصصة للمقيم العقاري ضمن نظام دراسة الحالة (إجادة). يقوم المقيم فيها برفع تقرير التقييم الصادر من برنامج **المقياس** الخارجي، وإدخال سعر التقييم، والإجابة على قائمة فحص، وتسجيل الملاحظات. بعد الإرسال تنتقل البيانات إلى أخصائي دراسة الحالة للمراجعة والاعتماد.

---

## الحقول والبيانات

### 1. تقرير التقييم (المقياس)

| المعرّف | النوع | الوصف | إلزامي |
|---------|-------|-------|--------|
| `evaluator_report_file` | `file` | ملف PDF صادر من برنامج المقياس | ✅ |

**القيود:**
- صيغة مقبولة: PDF فقط
- الحجم الأقصى: 20 MB
- ملف واحد لكل عقار

---

### 2. سعر التقييم

| المعرّف | النوع | الوصف | إلزامي |
|---------|-------|-------|--------|
| `evaluator_price` | `decimal` | قيمة العقار التقديرية بالريال السعودي | ✅ |

**ملاحظات:**
- يُخزَّن كرقم عشري (مثال: `1250000.00`)
- العملة ثابتة: ريال سعودي (SAR)

---

### 3. قائمة فحص المقيم

جميع أسئلة القائمة من نوع `boolean` (نعم / لا). الأسئلة التالية تحتوي على حقول تبعية تظهر عند الإجابة بـ **نعم**.

#### أسئلة بدون حقول تبعية

| المعرّف | نص السؤال |
|---------|-----------|
| `q_plan_match` | هل رقم المخطط مطابق للصك؟ |
| `q_excess_zoning` | هل القطعة زائدة تنظيمية؟ |
| `q_land_waqf` | هل الأرض موقوفة؟ |
| `q_property_waqf` | هل العقار موقوف؟ |
| `q_expropriation` | هل يوجد نزع على منطقة العقار؟ |
| `q_agriculture_inquiry` | هل تم الاستعلام من وزارة الزراعة حيال الأرض الزراعية؟ |
| `q_overlap` | هل يوجد تداخل في الأصل؟ |
| `q_shared_building` | هل يوجد على الأصل مبنى مشترك؟ |
| `q_environmental_factors` | هل هناك أي عوامل بيئية أو تنظيمية قد تؤثر على العقار؟ (مثل طريق مستقبلي أو قيود بناء) |
| `q_unregistered_additions` | هل العقار يحتوي على أي إضافات غير مسجلة في الصك؟ |

---

#### أسئلة بحقول تبعية

##### أ. هل الصك مشاع؟ (`q_shared_deed`)

عند الإجابة بـ **نعم** تظهر الحقول التالية:

| المعرّف | النوع | الوصف | إلزامي عند الظهور |
|---------|-------|-------|-------------------|
| `shared_deed_scope` | `enum` | نطاق الملكية: `full` (كامل المساحة) / `part` (جزء محدد) | ✅ |
| `shared_deed_percentage` | `string` | نسبة الملكية (مثال: `3/8` أو `37.5%`) — يظهر فقط إذا كانت قيمة `shared_deed_scope` = `part` | ✅ عند اختيار جزء |

**منطق الظهور:**
```
q_shared_deed = true
  └─ shared_deed_scope (مطلوب)
       └─ إذا shared_deed_scope = "part"
            └─ shared_deed_percentage (مطلوب)
```

---

##### ب. هل يوجد عقد إيجار؟ (`q_lease_exists`)

عند الإجابة بـ **نعم** يظهر الحقل التالي:

| المعرّف | النوع | الوصف | إلزامي عند الظهور |
|---------|-------|-------|-------------------|
| `q_lease_active` | `boolean` | هل عقد الإيجار ساري المفعول؟ | ✅ |

**منطق الظهور:**
```
q_lease_exists = true
  └─ q_lease_active (مطلوب)
```

---

##### ج. هل يوجد ملاحظات فنية قد تؤثر على قيمة العقار؟ (`q_technical_notes_exists`)

عند الإجابة بـ **نعم** يظهر الحقل التالي:

| المعرّف | النوع | الوصف | إلزامي عند الظهور |
|---------|-------|-------|-------------------|
| `technical_notes_text` | `text` | وصف الملاحظات الفنية | ✅ |

**منطق الظهور:**
```
q_technical_notes_exists = true
  └─ technical_notes_text (مطلوب)
```

---

### 4. الملاحظات على العقار

| المعرّف | النوع | الوصف | إلزامي |
|---------|-------|-------|--------|
| `evaluator_notes` | `text` | ملاحظات عامة على العقار لا تندرج ضمن قائمة الفحص | ❌ |

## قواعد التحقق (Validation)

| الحقل | القاعدة |
|-------|---------|
| `evaluator_report_file` | مطلوب · PDF فقط · حجم ≤ 20 MB |
| `evaluator_price` | مطلوب · رقم موجب · أكبر من صفر |
| جميع أسئلة `q_*` | مطلوب · قيمة واحدة: `true` أو `false` |
| `shared_deed_scope` | مطلوب إذا `q_shared_deed = true` |
| `shared_deed_percentage` | مطلوب إذا `shared_deed_scope = "part"` · نص حر (كسر أو نسبة مئوية) |
| `q_lease_active` | مطلوب إذا `q_lease_exists = true` |
| `technical_notes_text` | مطلوب إذا `q_technical_notes_exists = true` |

## سير العمل

```
المقيم يفتح النافذة
  │
  ├─ 1. يرفع ملف PDF من برنامج المقياس
  ├─ 2. يدخل سعر التقييم
  ├─ 3. يجيب على قائمة الفحص (13 سؤالاً)
  │       └─ الأسئلة ذات الحقول التبعية تُكمل حقولها عند الإجابة بنعم
  ├─ 4. يكتب الملاحظات العامة (اختياري)
  │
  └─ يضغط "إرسال للأخصائي"
        └─ التحقق من اكتمال جميع الحقول الإلزامية
              ├─ ناجح → تنتقل البيانات لأخصائي دراسة الحالة
              └─ فاشل → تُعرض رسالة خطأ مع الحقول الناقصة
```

---

## ملاحظات التنفيذ

- **حفظ تلقائي:** يُحفظ المسودة تلقائياً عند كل تعديل قبل الإرسال النهائي.
- **قفل البيانات:** بعد الإرسال تُقفل النافذة ولا يمكن التعديل إلا بإجراء إعادة فتح من الأخصائي.
- **الأصالة:** جميع البيانات المُدخلة من المقيم تُخزَّن تحت سجله بدور `أصيل` — لا يُعدّل الأخصائي قيم المقيم مباشرة، بل يُسجّل تعديلاته بشكل منفصل.
- **ارتباط بالعقار:** النافذة مرتبطة بـ `property_id` واحد؛ كل عقار في أمر العمل له سجل مستقل للمقيم.

## قرارات المنتج (نموذج أولي — واجهة فقط)

| الموضوع | القرار |
|---------|--------|
| تبويب نموذج الدراسة للمقيم | **نعم** — يملأ أسئلة النموذج المسندة لطرف `val` في المصفوفة فقط |
| إخفاء «لا دور» | الأسئلة غير المسندة **لا تظهر** للطرف (لا عرض فقط) |
| سعر الأخصائي | **استرشادي للقراءة** — مصدر القيمة هو المقيم وحده |
| قبل اكتمال المعاينة | سياق للقراءة فقط (بيانات العقار + مساهمات الأطراف) |
| بعد الإرسال | الحالة `submitted` — تختفي من قائمة المقيم |
| إرجاع من الأخصائي | `reopened` — تعديل كامل (PDF، السعر، القائمة، الملاحظات) |
| التخزين (النموذج الأولي) | `localStorage` — بدون API |

**حالات السجل:** `draft` → `submitted` → (`reopened`) → `completed`

**الشرط:** لا يُفعَّل إدخال التقييم إلا بعد اكتمال مهمة `field-inspection` لنفس العقار.
