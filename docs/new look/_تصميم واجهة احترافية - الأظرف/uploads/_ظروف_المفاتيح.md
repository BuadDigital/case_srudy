# ظروف المفاتيح — دليل النظام التفصيلي

> وثيقة مرجعية لوحدة **ظروف المفاتيح** كما هي مطبّقة في نظام إجادة للتقييم العقاري.
> تغطي منطق الأعمال، نموذج البيانات، الحالات، الواجهات، وواجهات البرمجة (API).

---

## ١. ما هو الظرف؟

**الظرف (ظرف المفاتيح / Key Envelope)** هو **وحدة التتبع والعهدة الأساسية** للمفاتيح في النظام — وليس المفتاح الفردي.

| مبدأ | التفصيل |
|---|---|
| وحدة العهدة | الظرف كاملاً، لا يُناول مفتاح بمفرده |
| مرجع الربط | **رقم الطلب من منصة إنفاذ** — وليس رقم أمر العمل الداخلي |
| ما يُكتب على الظرف فعلياً | رقم الطلب + عدد المفاتيح |
| محتوى الظرف | قد يشمل مفاتيح عقارات إجادة + مفاتيح لشركات أخرى؛ النظام يربط فقط عقارات إجادة بنفس رقم الطلب |
| هوية المفتاح | لا تُعرف إلا ميدانياً بتجربة الفتح على الصك (`matched` / `unmatched`) |

**المسؤول الأول:** المراجع الحكومي — يستلم من المحاكم ما يُمكّن فريق التقييم من دخول العقارات ودراستها، ويُوثَّق قانونياً عبر نفاذ كمسؤول عن الظرف.

---

## ٢. لماذا الظرف وليس المفتاح؟

1. المحكمة تُسلّم **ظرفاً واحداً** مرتبطاً بطلب إنفاذ، وليس مفاتيح مُعنونة لكل صك.
2. أي تسليم لاحق (للمعاين، لشركة أخرى، أو إرجاع للمحكمة) يحدث على **مستوى الظرف**.
3. الربط النهائي «مفتاح ↔ صك» يُؤكَّد فقط بعد التجربة الميدانية.
4. الأتعاب والإثباتات المالية (خطاب الاستلام + صورة الظرف) مرتبطة بعملية **استلام الظرف**، لا بكل مفتاح.

---

## ٣. مسارات الدخول للعقار (سياق الظرف)

الظرف أحد مسارات التمكين من الدخول، وليس المسار الوحيد:

```
هل العقار أرض (بدون مبنى)؟
       ↙                        ↘
      نعم                        لا
       ↓                          ↓
 لا مفاتيح              هل العقار مشغول؟
 يكمل مباشرة              ↙              ↘
                          لا              نعم
                           ↓                ↓
                    استلام ظرف      هل يوجد خطاب تمكين؟
                    مفاتيح من         ↙              ↘
                    المحكمة          نعم              لا
                                      ↓                ↓
                                 زيارة مباشرة   هل يوجد محظر إخلاء؟
                                 بدون مفتاح       ↙              ↘
                                                 نعم              لا
                                                  ↓                ↓
                                             معلّق حتى          تعذّر
                                             تنفيذ الإخلاء
```

> **قاعدة حاسمة:** خطاب التمكين ومحظر الإخلاء خاصان بالعقارات المشغولة، وهما **مساران مستقلان تماماً عن الظرف** (`PropertyCourtAccess`).

مستندات زيارة المحكمة التي يحملها المراجع:

| المستند | المصدر | النطاق |
|---|---|---|
| خطاب الإسناد | منصة إنفاذ | عقار واحد لكل خطاب |
| خطاب التفويض الداخلي | الشركة (من النظام) | قائمة عقارات الدائرة في الزيارة |

## ٤. سيناريوهات الاستلام

| الرمز | المعرّف في النظام | المعنى | ما يُشترط عند التسجيل |
|---|---|---|---|
| **أ** | `court` | المفاتيح موجودة بالمحكمة — استلام مباشر | عدد فعلي ≥ 1، خطاب استلام، صورة الظرف → **توليد أتعاب** |
| **ب** | `missing` | المفاتيح غير موجودة | أرقام تواصل (طالب التنفيذ / وكيله / أطراف) — **بدون أتعاب تلقائية** |
| **ج** | `third_party` | المفاتيح عند طرف آخر (مثلاً وكيل بيع) | خطاب من حامل المفتاح — **بدون أتعاب تلقائية** |

## ٥. حالات الظرف (Status)

| الحالة بالعربية | المعرّف | المعنى |
|---|---|---|
| بعهدة المراجع | `reviewer` | الظرف تحت مسؤولية المراجع مباشرة (الحالة الابتدائية بعد التسجيل) |
| بعهدة المعاين | `assessor` | سُلّم داخلياً للمعاين الميداني وأُكّد الاستلام |
| بعهدة طرف خارجي | `external` | سُلّم لشركة تقييم أخرى أو جهة خارج النظام |
| مُرجَع للمحكمة | `returned` | أُعيد للمحكمة بعد انتهاء الغرض أو التعذّر |

## ٦. دورة حياة الظرف

```
زيارة المحكمة
    ↓
استلام الظرف الفعلي (خطاب + صورة + توثيق نفاذ)
    ↓
تسجيل في النظام برقم الطلب → Status = reviewer
    ↓
[سيناريو أ فقط] توليد بند أتعاب (~350 ر.س، قابل للضبط)
    ↓
إسناد مبدئي اختياري: صك ↔ مفتاح (status = pending)
    ↓
مناولة داخلية → pending_confirm → تأكيد المعاين → Status = assessor
    ↓
الميدان: تجربة المفاتيح
    ↙                         ↘
 matched                    unmatched
    ↓                           ↓
متابعة الدراسة         تواصل / تعليق دخول / مسار بديل
    ↓
(اختياري) مناولة خارجية بخطاب → Status = external
    ↓
استرداد receive_back → Status = reviewer
    ↓
إرجاع للمحكمة return_court → Status = returned
```

## ٧. نموذج البيانات

الكيان الأساسي: `KeyEnvelope` — جدول في مخطط العمليات (`operations`).

### ٧.١ حقول الظرف

| الحقل | النوع | الوصف |
|---|---|---|
| `Id` | Guid | المعرّف |
| `RequestNumber` | string | رقم طلب إنفاذ — مرجع الظرف |
| `Court` / `Circuit` | string | المحكمة / الدائرة |
| `KeysCountLabeled` | int | العدد المكتوب على الظرف |
| `KeysCountActual` | int | العدد بعد العد — **المعتمد في النظام** |
| `CountMismatch` | computed | تنبيه عندما المكتوب ≠ الفعلي |
| `ReceiptAttachmentId` | Guid? | خطاب الاستلام |
| `PhotoAttachmentId` | Guid? | صورة الظرف (إثبات الأتعاب) |
| `ThirdPartyLetterAttachmentId` | Guid? | خطاب الطرف الثالث (سيناريو ج) |
| `ContactPhones` | string? | أرقام التواصل |
| `Notes` | string? | ملاحظات |
| `ReceiveScenario` | string | `court` \| `missing` \| `third_party` |
| `Status` | string | `reviewer` \| `assessor` \| `external` \| `returned` |
| `FeeGenerated` / `FeeAmountSar` | bool / decimal? | هل وُلد بند أتعاب ومبلغه |
| `CreatedBy*` / `CreatedAtUtc` / `UpdatedAtUtc` | | تدقيق |

**قاعدة تعارض العدد:** إذا اختلف العدّان، يُعرض تنبيه ويُطلب تعديل خطاب الاستلام في المحكمة فوراً، والنظام يحفظ الرقمين معاً دون منع التسجيل.

### ٧.٢ إسناد المفاتيح للصكوك (`KeyEnvelopeAssignment`)

مرحلتان:

| المرحلة | من | الحالة | المعنى |
|---|---|---|---|
| تسجيل مبدئي | المراجع (غالباً عند الاستلام) | `pending` | استرشادي فقط — لا يوقف العمل |
| تأكيد ميداني | المعاين بعد التجربة | `matched` أو `unmatched` | نجح الفتح أو فشل |

| الحقل | الوصف |
|---|---|
| `DeedNumber` | رقم الصك |
| `PropertyId` | ربط اختياري بعقار النظام |
| `Status` | `pending` \| `matched` \| `unmatched` |
| `ConfirmedBy*` / `ConfirmedAtUtc` | من أكّد ومتى |
| `Notes` | ملاحظات التجربة |

### ٧.٣ المناولة — سلسلة العهدة (`KeyEnvelopeHandoff`)

| النوع | المعرّف | الحالة التالية للظرف | خطاب رسمي |
|---|---|---|---|
| تسليم داخلي | `internal` | `assessor` (بعد التأكيد) | غير مطلوب — المراجع يسجّل + المعاين يؤكّد |
| تسليم خارجي | `external` | `external` | مطلوب (رقم + مرفق) |
| استلام من طرف | `receive_back` | `reviewer` | مطلوب |
| إرجاع للمحكمة | `return_court` | `returned` | مطلوب |

حالات المناولة: `pending_confirm` → `confirmed` / `completed`.

**لماذا التسليم الداخلي إلزامي في النظام؟** المراجع موثّق رسمياً عبر نفاذ كمسؤول عن الظرف؛ أي تسليم بدون تسجيل يُبقي المسؤولية القانونية معلّقة عليه.

### ٧.٤ سجل الحركات (`KeyEnvelopeTimelineEntry`)

أحداث موثّقة زمنياً، منها:

| `EventType` | المعنى |
|---|---|
| `created` | إنشاء الظرف |
| `assignment_added` | إضافة إسناد صك |
| `assignment_confirmed` | تأكيد مطابقة/عدم مطابقة |
| `handoff_created` | إنشاء مناولة |
| `handoff_confirmed` | تأكيد مناولة |
| `fee_generated` | توليد أتعاب |
| `status_changed` | تغيّر حالة العهدة |

### ٧.٥ أتعاب استلام المفاتيح (`KeyReceiptFeeCharge`)

| القاعدة | التفصيل |
|---|---|
| متى؟ | السيناريو **أ** (`court`) فقط عند اكتمال التسجيل بالمرفقات |
| المبلغ الافتراضي | **350 ر.س** — قابل للضبط عبر `PartyFeePricingConfig.KeyReceiptFeeSar` |
| الحالة | `open` (مفتوح للتحصيل) \| `collected` (تم التحصيل) |
| الإثبات | صورة الظرف + خطاب الاستلام مرتبطان ببند الأتعاب |
| التقرير | تقرير أتعاب من واجهة المفاتيح / API |

السيناريوهان ب وج **لا يولّدان** بند أتعاب تلقائياً.

### ٧.٦ مسار المحكمة المستقل (`PropertyCourtAccess`)

| الحقل / الحالة | المعنى |
|---|---|
| خطاب تمكين | دخول عقار مشغول بدون مفتاح → `enabled_no_key` |
| محظر إخلاء | تعليق الدراسة حتى الإخلاء → `suspended_eviction` |
| `none` | لا يوجد أثر على تعليق الدراسة من هذا المسار |

## ٨. بوابة حالة المفاتيح (Key Gate)

المكوّن `PropertyKeyGateResolver` يوحّد رؤية «هل المفاتيح متوفرة لهذا العقار؟» لواجهات المراجعة الحكومية والمعاينة.

**ترتيب الحسم:**

1. مسار المحكمة `enabled_no_key` (تمكين بدون مفتاح)
2. ظرف مرتبط برقم الطلب / الصك
3. بيانات قديمة من تسليم المراجعة الحكومية (legacy)
4. لا مصدر → `none`

حقول بارزة في الإسقاط:

| الحقل | قيم شائعة |
|---|---|
| `keysStatus` | `received` \| `pending` \| `not_required` \| `""` |
| `keyHandedToInspector` | `yes` \| `no` \| `""` |
| `source` | `envelope` \| `court_access` \| `legacy` \| `none` |
| `envelopeMissingWarning` | `true` إذا كانت البيانات القديمة تقول «مستلمة» ولا يوجد ظرف مسجّل |

في طابور المراجعة الحكومية يظهر شارة **«بانتظار الظرف»** عند التحذير؛ الإنهاء لا يُمنع، وقد يحدث مزامنة ناعمة للإسناد/المناولة على ظرف موجود إن وُجد.

## ١٠. واجهة البرمجة (API)

الخدمة: **Operations API** — المسار الأساسي ` /api/key-envelopes `  
الصلاحية الشائعة للتعديلات: `SubmitPartyWork`.

| الطريقة | المسار | الغرض |
|---|---|---|
| `GET` | `/api/key-envelopes` | قائمة الظروف |
| `GET` | `/api/key-envelopes/{id}` | التفاصيل (إسنادات + مناولات + timeline + عقارات مرتبطة) |
| `POST` | `/api/key-envelopes` | تسجيل ظرف جديد |
| `DELETE` | `/api/key-envelopes/{id}` | حذف |
| `GET` | `/api/key-envelopes/linked-properties?requestNumber=` | عقارات إجادة بنفس رقم الطلب |
| `POST` | `/api/key-envelopes/{id}/assignments` | إضافة إسناد صك |
| `POST` | `/api/key-envelopes/{id}/assignments/{assignmentId}/confirm` | تأكيد matched/unmatched |
| `POST` | `/api/key-envelopes/{id}/handoffs` | إنشاء مناولة |
| `POST` | `/api/key-envelopes/{id}/handoffs/{handoffId}/confirm` | تأكيد مناولة داخلية |
| `GET` | `/api/key-envelopes/fee-report` | تقرير الأتعاب |
| `POST` | `/api/key-envelopes/{id}/fee-collected` | تعليم الأتعاب كمحصّلة |
| `GET` | `/api/key-envelopes/gate?...` | بوابة حالة المفاتيح للعقار |
| `GET` | `/api/key-envelopes/court-access?requestNumber=` | قائمة تمكين/إخلاء |
| `PUT` | `/api/key-envelopes/court-access` | إنشاء/تحديث تمكين أو محظر |

عميل TypeScript: `packages/api-client` (وحدات الـ prototype / keys).

## ١١. العلاقات مع بقية النظام

| المفهوم | العلاقة |
|---|---|
| رقم طلب إنفاذ | مفتاح الربط الأساسي ظرف ↔ عقارات |
| أمر العمل / العقار | ربط عبر `RequestNumber` و/أو `PropertyId` + رقم الصك في الإسناد |
| مهمة المراجعة الحكومية | حقول مفاتيح قديمة + تحذير gate + مزامنة ناعمة عند الإنهاء |
| المعاينة الميدانية | تأكيد المناولة والإسناد؛ استهلاك الـ gate لتوفر المفتاح |
| المالية | `KeyReceiptFeeCharge` + تسعير `KeyReceiptFeeSar` |
| تعليق الدخول | مفتاح غير مطابق أو محظر إخلاء قد يولّدان تعليقاً على الدراسة |
| الرفع على النفاذ | قسم «استلام المفاتيح» في مواصفات الرفع — منفصل عن CRUD الظرف |

> ملاحظة تقنية: دوال باسم `HandleEnvelopeAsync` في مستهلكي الإشعارات تشير إلى **مغلف رسائل** (messaging)، وليست ظرف المفاتيح.

## ١٣. ملخص سريع للقواعد الحاكمة

1. **الظرف = وحدة العهدة**؛ المناولة دائماً للظرف كاملاً.
2. **رقم طلب إنفاذ** هو مرجع الربط — وليس أمر العمل الداخلي.
3. **الصورة + خطاب الاستلام** إلزاميان لسيناريو المحكمة (أ) وهما أساس الأتعاب.
4. **التسليم الداخلي** يُسجَّل ويُؤكَّد في النظام لإغلاق ثغرة المسؤولية.
5. **الإسناد المبدئي للصكوك** استرشادي حتى يؤكد المعاين في الميدان.
6. **التمكين ومحظر الإخلاء** مساران مستقلان عن الظرف.
7. **الأتعاب التلقائية** لسيناريو أ فقط (افتراضي 350 ر.س).
