# طبقة المهام (Tasks) — منطق النظام الكامل للمطوّر

مرجع تنفيذي شامل لطبقة المهام كما هي مطبّقة في `Case Study.html`. يغطّي نموذج البيانات، الحالات والانتقالات، الصلاحيات، منطق التذكير المرتبط بساعات العمل، الأولوية، الإسناد، خطاب التفويض، والواجهات. مكتوب للمطوّر لإعادة البناء على الواجهة الخلفية.

---

## 1. المفهوم

**المهمة** = عمل محدّد يُسنده منشئ (أخصائي دراسة الحالة أو مشرف القسم) إلى مستخدم منفّذ، بموعد استحقاق، وأولوية، ونطاق ربط.

مبدآن:
- **واجهة موحّدة** لإنشاء كل المهام (نافذة منبثقة).
- **نوع المهمة يحدّد المخرَج**: نوع «زيارة محكمة» يفعّل مولّد خطاب التفويض؛ أنواع أخرى قد تفعّل نماذج/مخرجات مختلفة.

المراجعة الحكومية وخطاب التفويض ليسا مسارين مستقلّين، بل **حالتان من هذه الطبقة**.

## 2. نموذج البيانات

```ts
type TaskType   = 'court_visit' | 'reshoot' | 'field_visit' | 'inquiry' | 'general';
type TaskScope  = 'transaction' | 'work_order' | 'multi' | 'general';
type TaskStatus = 'created' | 'in_progress' | 'paused' | 'completed' | 'cancelled';
type Priority   = 'high' | 'medium' | 'low';

interface Task {
  id: string;                 // مثل T-2041 (تسلسلي)
  type: TaskType;             // يحدّد المخرَج والأيقونة
  title: string;
  desc?: string;
  scope: TaskScope;
  // الربط حسب النطاق:
  deeds?: string[];           // أرقام الصكوك — transaction (عنصر واحد) | multi | work_order (مشتقّة)
  poNumber?: string;          // أمر العمل — work_order | transaction (مشتقّ من الصك)
  // الإسناد:
  assignee: string;           // معرّف المنفّذ
  createdBy: string;          // المنشئ (أخصائي/مشرف)
  // الزمن:
  createdAt: number;          // ISO/epoch
  dueAt: number;              // يوم + ساعة
  // الحالة:
  status: TaskStatus;
  prevStatus?: TaskStatus;    // يُحفظ عند الإيقاف المؤقت لاستئناف الحالة السابقة
  priority: Priority;
  // التتبّع:
  reminders?: { at: number; auto: boolean }[];   // سجل التذكيرات (يدوية/تلقائية)
  comments?: Comment[];       // خيط التحديثات
  // مخرَج court_visit فقط:
  ref?: string;               // الرقم المرجعي للخطاب، مثل خ.ت-2026-0041
  letterRows?: LetterRow[];   // لقطة (snapshot) بيانات الخطاب عند الإصدار
}

interface Comment {
  who: 'creator' | 'assignee' | 'system';
  at: number;
  text: string;
  files?: { name: string; size: string }[];
  kind?: 'comment' | 'close' | 'reminder' | 'update';
}

interface LetterRow {
  po: string; deed: string; owner: string;
  request: string; court: string; circuit: string;
}
```

## 3. نطاق الربط (Scope)

| النطاق | القيمة | الربط | الحقول المطلوبة |
|---|---|---|---|
| معاملة | `transaction` | صك واحد | `deeds[0]` (+ `poNumber` مشتقّ) |
| أمر عمل | `work_order` | كامل الـ PO | `poNumber` (+ `deeds` = كل صكوك الـ PO) |
| عدة معاملات | `multi` | مجموعة صكوك | `deeds` (صكّان فأكثر) |
| عامة | `general` | لا ربط | — |

- مهام `transaction` تُعرض في سجل المعاملة المرتبطة.
- مهام `general` مستقلة تماماً.
- عند `court_visit`: الافتراضي `work_order` أو `multi` (الخطاب يجمع صكوك محكمة+دائرة).

## 4. الحالات والانتقالات

```
منشأة (created) ──▶ قيد التنفيذ (in_progress) ──▶ مكتملة (completed)
      │  ▲                    │  ▲                       
      │  │                    │  │                       
      ▼  │ استئناف            ▼  │ استئناف                
   متوقفة مؤقتاً (paused) ◀────┘                          
      │                                                   
      └──────────────┬──────────────┐                     
   (أي حالة نشطة) ───▶ ملغاة (cancelled)                   
```

| الحالة | المعنى | من يغيّرها | الانتقالات المتاحة |
|---|---|---|---|
| `created` | أُسندت ولم تبدأ | — | → in_progress (بدء) · → paused (إيقاف) · → cancelled |
| `in_progress` | المنفّذ بدأ العمل | المنفّذ | → completed (إغلاق) · → paused · → cancelled |
| `paused` | موقوفة مؤقتاً بانتظار طرف آخر | المنشئ/المشرف | → prevStatus (استئناف) · → cancelled |
| `completed` | أُنجزت | المنفّذ | نهائية |
| `cancelled` | أُلغيت | المنشئ/المشرف | نهائية |

**تعريفات مشتقّة (مهمة للفلترة والمنطق):**
- `TERMINAL` = { completed, cancelled } — حالات نهائية.
- `active(task)` = status ∈ { created, in_progress } — تظهر افتراضياً، ولها عدّاد وتذكير.
- `paused` ليست نهائية لكنها **مخفيّة افتراضياً** (كالنهائية) ويتوقف لها العدّاد والتذكير.

**الإيقاف المؤقت:** عند `pause` يُحفظ `prevStatus = status` ثم `status = 'paused'`؛ وعند `resume` يعود `status = prevStatus || 'in_progress'`.

## 5. الأولوية (Priority)

| الأولوية | القيمة | اللون | مدة الاستحقاق الافتراضية عند الإنشاء |
|---|---|---|---|
| عالية | `high` | `#d9694f` أحمر | ٤ ساعات |
| متوسطة (افتراضي) | `medium` | `#d9a441` أصفر | **نصف يوم** (١٢ ساعة) |
| منخفضة | `low` | `#8a8d96` رمادي | يوم كامل |

- الافتراضي عند إنشاء مهمة جديدة: **متوسطة**، وموعد استحقاق = الآن + نصف يوم.
- تغيير الأولوية متاح لاحقاً (إجراء «تغيير الأولوية») مع خيار إعادة ضبط الاستحقاق يدوياً أو حسب مدة الأولوية الجديدة.
- تغيير الأولوية **يعيد حساب وتيرة التذكير تلقائياً**.

## 6. منطق التذكير المرتبط بساعات العمل

**ساعات العمل:** ٨:٠٠ ص – ٥:٠٠ م. **الظهر (نقطة مراجعة):** ١٢:٠٠. **أيام العمل:** الأحد–الخميس (`getDay()` من 0=الأحد إلى 4=الخميس؛ الجمعة/السبت عطلة).

**نموذج المراجعة الافتراضي:** أغلب الأوامر تُعالَج ضمن نصف ساعات العمل — ما يطرأ صباحاً يُراجَع الظهر (١٢م)، وما يطرأ بعد الظهر يُراجَع نهاية الدوام (٥م).

### وتيرة التذكير حسب الأولوية

| الأولوية | الوتيرة | حساب «التذكير القادم» |
|---|---|---|
| عالية | كل ساعة **ضمن الدوام** | `nextWorkHour(now)` |
| متوسطة | عند نقطة المراجعة (منتصف/نهاية الدوام) | `nextCheckpoint(now)` |
| منخفضة | مراجعة يوم العمل التالي | `nextWorkDayNoon(now)` |

### الدوال المرجعية

```js
WH_START = 8; WH_END = 17; WH_NOON = 12;
isWorkDay(d)      = d.getDay() >= 0 && d.getDay() <= 4          // الأحد–الخميس
atHour(d, h)      = طابع زمني لليوم d عند الساعة h:00
nextWorkDayNoon(ts) = ظهر أول يوم عمل تالٍ بعد ts
nextCheckpoint(ts):
    إذا يوم عمل و الساعة < 12  → ظهر اليوم (12:00)
    إذا يوم عمل و الساعة < 17  → نهاية اليوم (17:00)
    غير ذلك                   → nextWorkDayNoon(ts)
nextWorkHour(ts):
    إذا يوم عمل و الساعة < 8   → اليوم 08:00
    وإلا الساعة التالية إن كانت ضمن [8..17] ويوم عمل
    وإلا 08:00 من أول يوم عمل تالٍ
```

- العدّاد الحيّ في التفاصيل يعرض المدة حتى `nextReminderTs(task)` ويُحدَّث كل ثانية.
- **لا تذكيرات خارج الدوام أو في العطلة** — النقطة القادمة دائماً ضمن ساعات العمل.
- التذكير لا يعمل للحالات `paused` والنهائية.

### التذكير اليدوي والجماعي
- **تذكير فوري** (زر داخل الصف/التفاصيل، أو جماعي على المحدَّد): يضيف `reminders[] {at, auto:false}`، يسجّل تعليق `kind:'reminder'`، ويُطلق إشعاراً آنياً (toast) للمنفّذ.
- **التذكير الجماعي**: يُطبّق على المهام النشطة فقط ضمن المحدَّد (يتجاهل المتوقفة/النهائية).

## 7. الإسناد وإعادة التوجيه

- **الإسناد** عند الإنشاء إلزامي (`assignee`).
- **إعادة توجيه وإسناد**: إجراء يعيد إسناد المهمة لمنفّذ آخر، مع:
  - ضبط موعد تسليم جديد (تاريخ + وقت).
  - **سبب توجيه إلزامي** (يُمنع الحفظ بدونه).
  - تسجيل تلقائي في الخيط: «➤ من «X» إلى «Y» — موعد التسليم … السبب: …».

## 8. الصلاحيات

| العملية | الصلاحية |
|---|---|
| إنشاء مهمة | أخصائي دراسة الحالة، مشرف القسم |
| بدء التنفيذ / الإكمال | المنفّذ المكلَّف |
| إيقاف مؤقت / استئناف | المنشئ، المشرف |
| تغيير الأولوية / إعادة التوجيه | المنشئ، المشرف |
| إلغاء | المنشئ، المشرف |
| تذكير | المنشئ، المشرف |
| تعليق/مرفق | الطرفان (المنشئ + المنفّذ) |
| عرض | حسب الربط (أطراف المعاملة/الـ PO) + المنشئ + المنفّذ |

## 9. خيط التحديثات والمرفقات

- خيط زمني بين **المنشئ والمنفّذ**؛ كل تحديث: الاسم، الدور، الوقت، النص، ومرفقات اختيارية.
- رفع مستندات من **الطرفين**.
- أنواع القيود (`kind`):
  - `comment`: تحديث/استفسار عادي (فقاعة باسم الطرف).
  - `close`: تعليق إغلاق (يُوسم «تعليق إغلاق»).
  - `reminder`: إدخال نظام مضغوط (🔔) عند التذكير.
  - `update`: إدخال نظام مضغوط (⚑/➤/⏸/▶) عند تغيير الأولوية/الاستحقاق/التوجيه/الإيقاف/الاستئناف.
- **الإغلاق عبر نافذة**: عند الإكمال تُفتح نافذة لتعليق الإنجاز + رفع ملفات؛ عند التأكيد تتحوّل الحالة إلى `completed` ويُسجَّل القيد `kind:'close'`.

## 10. خطاب التفويض (مخرَج مهمة زيارة المحكمة)

يُنشأ **عبر مهمة** من نوع `court_visit`، لا مساراً مستقلاً:
- **المنشئ:** أخصائي دراسة الحالة/المشرف. **المنفّذ:** المراجع الحكومي.
- **النطاق:** `work_order` أو `multi`.
- **المخرَج:** يُولَّد تلقائياً عند إنشاء المهمة إن وُجدت صكوك مرتبطة:
  - `ref` = ترميز مرجعي موحّد تسلسلي: `خ.ت-2026-NNNN` (٤ خانات).
  - `letterRows` = **لقطة (snapshot)** لبيانات الصكوك وقت الإصدار.
  - الجدول (مسطّح، محاذاة الرؤوس مع المحتوى): **م | أمر العمل | رقم الصك | المالك | رقم الطلب | المحكمة / الدائرة**.
  - مفتاح التجميع المرجعي = المحكمة + الدائرة.
  - قابل للطباعة على الترويسة الرسمية (اسم النظام، الرقم المرجعي، التاريخ، جدول، توقيع المُعِدّ).
- **الفرق عن السابق:** الأخصائي يُنشئ **مهمة زيارة محكمة** للمراجع، والخطاب مُخرَج منها — لا يُنشأ من واجهة المراجعة الحكومية مباشرة.

## 11. الواجهات

### سجل المهام (القائمة)
- شريط KPI: مهام نشطة / منشأة / قيد التنفيذ / مكتملة.
- فلاتر: بحث + حالة + نطاق + زر **«إظهار جميع المهام»** (الافتراضي: النشطة فقط، وتُخفى المتوقفة والنهائية).
- **مربعات اختيار** لكل مهمة نشطة + تحديد الكل + شريط إجراء جماعي (تذكير المحدَّد دفعة واحدة).
- الأعمدة: [✔] | المهمة (أيقونة النوع + العنوان + المعرّف·النوع·الأولوية) | النطاق/الربط | المنفّذ | **الاستحقاق (عدّاد حيّ + مؤشّر قرب)** | الحالة | إجراءات (تذكير سريع + قائمة ⋮).
- **الترتيب:** النشطة (بالأقرب استحقاقاً، المتأخرة أعلى) ← المتوقفة ← النهائية.

### مؤشّر قرب الاستحقاق (دائرة)
| الحالة | الشرط (المتبقّي حتى الاستحقاق) | اللون |
|---|---|---|
| مريح | أكثر من ٨ ساعات | 🟢 `#3f8f5f` |
| يقترب | ٨ ساعات فأقل | 🟡 `#d9a441` |
| عاجل/متأخر | ساعتان فأقل أو متأخر | 🔴 `#d9694f` **وامض** |

### عدّاد الاستحقاق
- صيغة حيّة: `[N يوم ·] HH:MM:SS`، ويعرض «متأخرة» بالأحمر عند التجاوز.
- **بالون عند التمرير** يعرض التاريخ/الوقت الكامل.
- المتوقفة تعرض «متوقفة»؛ النهائية تعرض «—».

### إنشاء مهمة (نافذة موحّدة)
حقول: النوع* · المنفّذ (مُسندة إلى)* · العنوان* · الوصف · نطاق الربط* (مقسّم + حقول ربط شرطية) · الأولوية* (افتراضي متوسطة) · موعد الاستحقاق* (أزرار سريعة اليوم/غد/بعد غد + تاريخ + وقت). عند اختيار `court_visit` تظهر معاينة حيّة لخطاب التفويض.

### تفاصيل المهمة
رأس (عنوان + نوع + حالة + مرجع الخطاب) · **مؤشّر حالة (stepper)** · بطاقة التذكير التلقائي (وتيرة + عدّاد قادم + زر «تذكير الآن») · شبكة بيانات (المنفّذ، المنشئ، النطاق، الأولوية، تاريخ الإنشاء، الاستحقاق) · الوصف · قسم خطاب التفويض (لـ court_visit) · أزرار الإجراءات · خيط التحديثات والمرفقات.

## 12. ملاحظات للمطوّر

- **الاختصار = نفس الكيان:** الاختصارات السياقية داخل المعاملات مدخل مُعبّأ مسبقاً لواجهة المهمة الموحّدة (لا كيان منفصل).
- **نوع المهمة يحدّد المخرَج:** `court_visit` → مولّد خطاب التفويض؛ أنواع أخرى قد تُفعّل نماذج مختلفة.
- **الربط متعدّد:** `deeds` مصفوفة لدعم `multi`؛ في `work_order` تُشتقّ من صكوك الـ PO؛ في `transaction` صك واحد + PO مشتقّ.
- **snapshot الخطاب:** خزّن `letterRows` وقت الإصدار ولا تعِد اشتقاقها لاحقاً (لضمان ثبات المستند).
- **التذكير خادم-جانبي:** حوّل منطق `nextReminderTs` (المرتبط بالدوام) إلى مجدول (scheduler) على الخادم يُطلق إشعارات فعلية عبر قناة النظام؛ الواجهة الحالية تحاكيه بعدّاد + toast.
- **الإشعار:** كل إسناد/تذكير/إعادة توجيه يُشعر المنفّذ.
- **التقويم الهجري/الميلادي:** أيام العمل والنقاط الزمنية مبنيّة على التقويم الميلادي وأيام الأحد–الخميس؛ اضبطها حسب سياسة المنشأة والإجازات الرسمية.
