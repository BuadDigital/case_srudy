# أوامر العمل (PO) — إجادة للتقييم

واجهة React + Tailwind CSS لشاشة «أوامر العمل» بنظام RTL كامل وألوان الشعار (كحلي `#12284C` وذهبي `#a4906f`).

## التشغيل

```bash
cd react-app
npm install
npm run dev
```

ثم افتح الرابط الذي يظهر (عادةً http://localhost:5173).

للبناء للإنتاج:

```bash
npm run build
npm run preview
```

## بنية الملفات

```
react-app/
├── index.html            # جذر الصفحة + استيراد خط Tajawal
├── package.json          # الاعتماديات (React 18 + Vite + Tailwind)
├── vite.config.js
├── tailwind.config.js    # الألوان والخط المخصص
├── postcss.config.js
└── src/
    ├── main.jsx          # نقطة الدخول
    ├── index.css         # توجيهات Tailwind + تنسيق شريط التمرير
    ├── data.js           # بيانات أوامر العمل + دوال مساعدة (الفرز/الألوان)
    ├── icons.jsx         # أيقونات SVG مضمّنة (بدون مكتبات خارجية)
    ├── Logo.jsx          # شعار إجادة (SVG)
    ├── Sidebar.jsx       # القائمة الجانبية الكحلية
    ├── Header.jsx        # الهيدر (البحث + المستخدم)
    ├── KpiBand.jsx       # شريط المؤشرات
    ├── OrdersTable.jsx   # الجدول + صفوفه
    └── App.jsx           # التجميع + حالة البحث/الفلترة/الفرز
```

## المزايا

- **بحث** فوري (رقم PO، الصك، نوع الإسناد، اسم الأخصائي) — متزامن بين حقلي الهيدر والشريط.
- **فلاتر** الحالة ونوع الإسناد.
- **فرز** بالنقر على رؤوس الأعمدة (العقارات، المكتملة، التقدم، التواريخ) تصاعدي/تنازلي.
- أشرطة تقدّم وشارات حالة ملوّنة، وصور رمزية للأخصائيين.

## ربط ببيانات فعلية

استبدل مصفوفة `ORDERS` في `src/data.js` بنتيجة طلب API (مثلاً عبر `useEffect` + `fetch` في `App.jsx`)، مع الحفاظ على نفس أسماء الحقول.
