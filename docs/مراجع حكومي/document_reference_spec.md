# منطق الترقيم المرجعي للمستندات

نظام إجادة | ترميز الخطابات والتقارير الصادرة

---

## 1. البنية العامة

```
[الإدارة] - [النوع] - YYMMDD - NNN
```

مثال:
```
DS-LT-260719-001
```

| المقطع | الوصف | مثال |
|---|---|---|
| الإدارة | رمز الإدارة المُصدِرة (حرفان) | DS |
| النوع | رمز نوع المستند (حرفان) | LT |
| YYMMDD | التاريخ الميلادي: سنة/شهر/يوم | 260719 |
| NNN | عدّاد تسلسلي، ثلاث خانات (001–999) | 001 |

الفاصل: شرطة `-` بين كل مقطع.

---

## 2. رموز الإدارات

| الإدارة | الرمز |
|---|---|
| قسم دراسة الحالة | `DS` |
| قسم التقييم العقاري | `VL` |
| المالية والعقود | `FN` |
| الموارد البشرية | `HR` |
| الإدارة العامة | `GM` |

## 3. رموز أنواع المستندات

| النوع | الرمز |
|---|---|
| خطاب (تفويض وغيره) | `LT` |
| تقرير دراسة الحالة | `CS` |

> أنواع/إدارات جديدة تُضاف بنفس النمط (حرفان).

---

## 4. قواعد العدّاد

1. **النطاق:** عدّاد مستقل لكل تركيبة `[الإدارة][النوع]`.
   - خطاب دراسة الحالة (`DS-LT`) له عدّاده، وتقريرها (`DS-CS`) له عدّاد آخر، وخطاب التقييم (`VL-LT`) عدّاد ثالث — لا تتعارض.
2. **التصفير:** يبدأ من `001` مع كل يوم تقويمي جديد.
3. **التقويم:** ميلادي (يحكم كلاً من التاريخ في الرقم وتصفير العدّاد).
4. **الترتيب YYMMDD:** سنة ثم شهر ثم يوم — يضمن أن الفرز النصي = الفرز الزمني.
5. **عدم إعادة الاستخدام:** الرقم المُصدَر لا يُعاد استخدامه حتى عند إلغاء المستند.

---

## 5. أمثلة

```
DS-LT-260719-001    أول خطاب من دراسة الحالة، 19/07/2026
DS-LT-260719-002    ثاني خطاب من دراسة الحالة، نفس اليوم
DS-CS-260719-001    أول تقرير دراسة حالة، نفس اليوم (عدّاد مستقل)
VL-LT-260719-001    أول خطاب من التقييم العقاري، نفس اليوم
DS-LT-260720-001    أول خطاب من دراسة الحالة، اليوم التالي (تصفّر)
```

---

## 6. ملاحظات للمطور

- **التخزين:** يُحفظ الرقم كاملاً كنص، مع تخزين مقاطعه منفصلة (`dept`, `type`, `date`, `seq`) لتسهيل الفهرسة والفرز والبحث.
- **العدّاد الذرّي:** توليد `seq` يجب أن يكون عملية ذرّية (atomic) لمنع التكرار عند الإصدار المتزامن — استخدم قفل/معاملة على مفتاح `(dept, type, date)`.
- **snapshot:** الرقم المرجعي يُثبَّت في سجل المستند لحظة الإصدار ولا يتغير لاحقاً.
- **التوافق مع الأرشيف القديم:** النموذج المطبوع السابق يستخدم صيغة `S-05xx`. اعتماد الصيغة الجديدة يتطلب تأكيد الإدارة، مع الاحتفاظ بالأرقام القديمة كما هي في الأرشيف دون تحويل.

## 7. TypeScript

```typescript
type DeptCode = 'DS' | 'VL' | 'FN' | 'HR' | 'GM';
type DocType  = 'LT' | 'CS';

interface DocumentReference {
  full: string;        // "DS-LT-260719-001"
  dept: DeptCode;
  type: DocType;
  date: string;        // "260719" (YYMMDD ميلادي)
  seq: number;         // 1..999
  issuedAt: string;    // ISO timestamp
}

function buildReference(dept: DeptCode, type: DocType, date: Date, seq: number): string {
  const yy = String(date.getFullYear()).slice(-2);
  const mm = String(date.getMonth() + 1).padStart(2, '0');
  const dd = String(date.getDate()).padStart(2, '0');
  const nnn = String(seq).padStart(3, '0');
  return `${dept}-${type}-${yy}${mm}${dd}-${nnn}`;
}
```
