# مواصفات نظام دراسة الحالة — قاموس الحقول المركزي ونموذج الإسناد

> وثيقة موجَّهة لمساعد البرمجة (Cursor) لفهم منطق النظام وبنائه. تصف **المنطق الكامل**: الكيانات، الخصائص، القواعد، والسلوكيات.

## 0. نطاق الوثيقة

- **داخل النطاق:** نموذج البيانات، قواعد التحقق، السلوكيات، والعلاقات بين الكيانات.
- **خارج النطاق:** ثيم/شكل الشاشات وتنسيقها البصري (متّفق عليه مسبقاً بشكل منفصل). لا تستنتج التصميم البصري من هذه الوثيقة.
- النموذج التفاعلي `قاموس_الحقول_المركزي.html` مرجع بصري/سلوكي مساعد فقط، وليس مصدر التصميم النهائي.

## 0.1 تعليمات الدمج في النظام (مهم)

عند حقن الأداة/المنطق في النظام الفعلي، نفّذ بالترتيب:

1. **امسح الحقول التجريبية** في الأداة (FLD-001 … FLD-008) — بيانات عرض فقط؛ يبدأ القاموس فارغاً.
2. **اجلب حقول النظام الفعلية إلى الأداة** من مصدرها الرسمي (مخطط قاعدة البيانات / النماذج القائمة / واجهة برمجية)، وحوّل كل عمود/حقل إلى كيان `Field` بخصائصه (§3).
3. **خطّط الشاشات الحالية وادمجها**: أنشئ سجل الشاشات من شاشات النظام القائمة، واستنبط الإسنادات (أي حقلٍ يظهر في أي شاشة، لأي دور، وبأي طريقة استخدام) ودوّنها كـ `Assignment`.

ملاحظات حاكمة:

- استبدل القوائم الثابتة (الأدوار، الشاشات، `ROLE_SCREENS`، `SCREEN_STATUS`، الوسوم) بمصادر النظام الفعلية، أو بطبقة بيانات تقرأ منها.
- احتفظ **بالمنطق والقواعد والبنية** فقط، لا بالبيانات التجريبية.
- **حدّد مصدر الحقيقة لكل جلب:** من أين تُقرأ الحقول والشاشات (قاعدة البيانات؟ كود الواجهات؟ ملف يُزوَّد؟). لا يمكن «الجلب» دون مصدر معرّف.

## 1. المفهوم

قاموس حقول مركزي (Data Dictionary): **كل حقل يُعرَّف مرة واحدة فقط** بخصائصه، ثم يُصنَّف بالوسوم ويُسنَد للشاشات عبر الأدوار. الهدف منع تكرار تعريف الحقول وتوحيد مصدر الحقيقة.

المبدأ الحاكم: **سلوك الحقل يتبع سياقه (الدور والشاشة)، لا تعريفه.** فالإلزام والموثوقية وطريقة الاستخدام كلها خصائص للإسناد، لا للحقل.

## 2. الهرمية

```
النظام → الأدوار → الشاشات → الحقول
```

- **الدور (Role):** جهة/وظيفة في النظام (المكتب الهندسي، المالية، المعاين، …).
- **الشاشة (Screen):** نموذج/صفحة تخدم دوراً أو أكثر (علاقة متعدد-لمتعدد).
- **الحقل (Field):** أصغر وحدة معلومة، معرَّف مركزياً.
- **الإسناد (Assignment):** علاقة تربط الحقل بدورٍ وبشاشاتٍ تابعة لذلك الدور، وتحمل سلوك الحقل في ذلك السياق.

**محتوى الشاشة (حقولها) مشتقٌّ من الإسنادات** — نفس مصفوفة الإسناد تُقرأ من جهة الحقل («يظهر في هذه الشاشات») ومن جهة الشاشة («تحوي هذه الحقول»).

## 3. الكيانات (نموذج البيانات)

```typescript
// الحقل — يُعرَّف مرة واحدة في القاموس
interface Field {
  id: string;              // معرّف داخلي
  ref: string;             // الرقم المرجعي، فريد، يُعرض مسبوقاً بـ # (مثل FLD-001)
  name: string;            // التسمية الظاهرة
  type: FieldType;         // نوع البيانات (انظر §4)
  tags: string[];          // وسوم متعددة من قائمة معرّفة (انظر §5)
  source?: string;         // مصدر القيم — للقوائم/المرجع/الأرقام (انظر §6)
  parent?: string;         // الحقل الأب — للقائمة المرتبطة فقط (cascading)
  child?: string;          // الحقل الابن المعتمد عليه (إن وُجد)
  persisted: boolean;      // هل سُجّل/استُخدم في قاعدة البيانات؟ (يحكم الحذف، §10)
  assignments: Assignment[];
}

// الإسناد — يحمل سلوك الحقل لدورٍ معيّن في شاشاته
interface Assignment {
  role: string;            // الدور
  screens: string[];       // شاشات هذا الدور فقط (مجموعة فرعية من شاشاته)
  mode: "input" | "view";  // طريقة الاستخدام: إدخال أو عرض فقط
  required?: boolean;      // إلزامي — ذو معنى فقط عندما mode === "input"
  final?: boolean;         // الدور المعتمد (القيمة النهائية) — فقط عندما mode === "input"
}

interface Role { name: string; }

interface Screen {
  name: string;
  roles: string[];                 // الأدوار التي تخدمها (متعدد-لمتعدد)
  status: "موجودة" | "مخططة";      // مبنية فعلاً أم مخططة (§9)
}
```

## 4. أنواع الحقول (FieldType)

«النوع» = نوع البيانات فقط. **لا يُخلط مع الموثوقية** (متعدد المراحل ليست نوعاً، بل وضع موثوقية مشتق، §8).

```typescript
type FieldType =
  // نصوص
  | "text" | "textarea" | "richtext"
  // أرقام
  | "number" | "decimal" | "currency" | "percent"
  // تاريخ ووقت
  | "date" | "time" | "datetime"
  // اختيارات
  | "list" | "multiselect" | "radio" | "checkbox" | "linked" | "bool"
  // هوية واتصال
  | "email" | "phone" | "url"
  // متخصصة
  | "file" | "image" | "signature" | "geo" | "barcode" | "rating" | "color"
  // علائقية ومحسوبة
  | "relation" | "formula" | "autonum";
```

التسميات العربية: نص قصير، نص طويل، نص منسّق، رقم، عدد عشري، عملة، نسبة مئوية، تاريخ، وقت، تاريخ ووقت، قائمة منسدلة، اختيار متعدد، أزرار اختيار، مربعات تأشير، قائمة مرتبطة، نعم/لا، بريد إلكتروني، هاتف، رابط، مرفق/ملف، صورة، توقيع، موقع جغرافي، باركود/QR، تقييم، لون، مرجع لسجل، حقل محسوب، ترقيم تلقائي.

## 5. الوسوم (Tags)

- قائمة وسوم معرّفة مركزياً؛ المستخدم **يختار** منها ولا يكتب نصاً حراً.
- الحقل يحمل **عدة وسوم** (علاقة متعدد-لمتعدد).
- الوسم يصف سياق الحقل (مالي، هندسي، مشترك، إداري…)، ويُستخدم للتصنيف والفلترة فقط — **لا يحدد مكان ظهور الحقل** (ذلك دور الإسناد).
- إنشاء وسم جديد صلاحية إدارية؛ يُضاف للقائمة العامة ويصبح متاحاً للفلترة.

## 6. القوائم والمصدر والارتباط التتابعي

- **مصدر القيم (`source`)** مطلوب للأنواع: list, multiselect, radio, checkbox, linked, relation, number, decimal, currency, percent. (جدول مرجعي، وحدة، قائمة ثابتة…)
- **القائمة المرتبطة (`linked`)** قائمة فرعية تعتمد على حقلٍ أب (`parent`): قيمها تتغيّر حسب اختيار الأب (مثل: الحي يعتمد على المدينة). تُعرَّف كعلاقة بين حقلين، لا كحقل مستقل.

## 7. نموذج الإسناد (الحقل ← الدور ← الشاشة)

- الحقل يُسنَد لدورٍ أو أكثر؛ كل إسناد دور يحمل شاشةً أو أكثر.
- **علاقة متعدد-لمتعدد:** الشاشة قد تخدم أكثر من دور (مثل «أتعاب المكتب الهندسي» للمكتب الهندسي وللمالية معاً).
- **الشاشات فرعية من الدور:** عند الإسناد، تُعرض شاشات الدور المختار فقط (`Screen.roles` تحدد ذلك). تغيير الدور يُحدّث قائمة الشاشات ويُلغي أي شاشة لم تعد ضمن الدور الجديد.

### سلوك الإسناد (لكل assignment)

| الخاصية | المعنى | متى تكون ذات معنى |
|---|---|---|
| `mode` | إدخال (قابل للتعديل) أو عرض فقط (للقراءة) | دائماً |
| `required` | إلزامي | فقط عند `mode === "input"` |
| `final` | الدور المعتمد (صاحب القيمة النهائية) | فقط عند `mode === "input"` |

- عند `mode === "view"`: تُخفى/تُلغى `required` و`final` لأنهما بلا معنى — الدور هنا يعرض قيمةً أنتجها غيره ولا يُدخلها.

## 8. الموثوقية (Reliability) — أهم منطق

### تمييز الطبقات

- **«أولي / معتمد» حالةُ قيمةٍ** وقت التشغيل، **وليست تصنيفاً للحقل**. (لا نعرّف حقلين لمعلومة واحدة.)
- في القاموس نعرض **«وضع الموثوقية» المشتق**: `واحد` أو `متعدد`.

### قواعد الاشتقاق (حاسمة)

1. الموثوقية يُنتجها **الإدخال فقط**. أدوار «عرض فقط» لا تُحتسب في الاشتقاق.
2. ليكن `inputs` = الإسنادات التي `mode === "input"`.
3. **«أولي» لا معنى لها إلا بوجود «معتمد» تتجاوزها.** فالمصدر الوحيد = معتمد بالضرورة.
4. وضع الحقل:
   - `متعدد` ⟺ يوجد بين `inputs` دورٌ `final === true` **و** دورٌ آخر `final !== true`.
   - وإلا فهو `واحد`.
5. موثوقية إسناد إدخالٍ معيّن:
   - إن لم يوجد أي `final` بين `inputs` ⟸ **معتمد** (كل المصادر موثوقة، الوضع `واحد`).
   - إن وُجد `final`: الدور `final` ⟸ **معتمد**، وبقية أدوار الإدخال ⟸ **أولي** (الوضع `متعدد`).
6. **القيمة المعتمدة** للحقل المتعدد = القيمة التي أدخلها الدور `final`. وما قبلها قيم أولية قابلة للتجاوز.

```typescript
function inputAsgs(f: Field) {
  return f.assignments.filter(a => (a.mode ?? "input") === "input");
}
function fieldMode(f: Field): "واحد" | "متعدد" {
  const ins = inputAsgs(f);
  const anyFinal = ins.some(a => a.final);
  const anyNot   = ins.some(a => !a.final);
  return (anyFinal && anyNot) ? "متعدد" : "واحد";
}
function asgReliability(f: Field, a: Assignment): "أولي" | "معتمد" {
  const anyFinal = inputAsgs(f).some(x => x.final);
  if (!anyFinal) return "معتمد";
  return a.final ? "معتمد" : "أولي";
}
```

### العرض

- وسم الموثوقية (أولي/معتمد) داخل بطاقة الدور يظهر **فقط** للحقل `متعدد` ولأدوار الإدخال.
- الحقل `واحد` لا يُظهر موثوقيةً في الأدوار (هي معتمدة دائماً).
- الحقل المتعدد قد يُرافقه عرض «القيمة المعتمدة + سجل القراءات حسب ترتيب المرجعية» مع قياس التباين بين الأدوار (لقياس دقة كل دور).

## 9. سجل الشاشات والحالة

- لكل شاشة **حالة:** `موجودة` (مبنية) أو `مخططة` (لم تُبنَ بعد).
- **يُسمح بإسناد الحقول لشاشة مخططة** وتُميَّز بصرياً (وسم «مخططة»). فالقاموس توثيقٌ للموجود ومواصفاتٌ للقادم.
- الشاشات مصدرها **سجل الشاشات** (وليس ترميزاً ثابتاً): كل شاشة باسمها وأدوارها وحالتها.

### منهجية الإدخال (نظام قائم جزئياً)

1. **جرد الموجود:** أدرج كل شاشة قائمة مع دورها وحقولها الحالية، بحالة `موجودة`؛ وأي حقل غير معرّف يُضاف للقاموس (تعبئة عكسية).
2. **أضف الشاشات المخططة** بحالة `مخططة` مربوطةً بأدوارها.
3. **صمّم الإسناد للجميع:** توثيقٌ للموجودة، ومواصفاتٌ مسبقة للمخططة.
4. **عند بناء شاشة مخططة:** استخرج قائمة حقولها من القاموس كمواصفات بناء، ابنِها، ثم حوّل حالتها إلى `موجودة`.

## 10. الرقم المرجعي والحذف

- **الرقم المرجعي (`ref`)** فريد، يُولَّد تلقائياً للحقل الجديد، ويُعرض مسبوقاً بـ `#`، وقابل للنسخ بالضغط. لا يُخزَّن الرمز `#` ضمن القيمة.
- **الحذف:**
  - حقل **غير مستخدم/غير مسجّل** (`persisted === false`): يجوز حذفه نهائياً ويتحرّر رقمه المرجعي.
  - حقل **مستخدم/مسجّل** (`persisted === true`): **لا يُحذف**؛ يُعطَّل/يُؤرشَف، ويبقى رقمه المرجعي محفوظاً للأبد (حفاظاً على سلامة البيانات والسجل التاريخي).

## 11. الفلترة

أوجه الفلترة (تجمع بـ AND بين الأوجه، وبـ OR داخل الوجه):

- النوع (`type`)
- الوسم (`tag`)
- الدور (`role`) — من إسنادات الحقل
- الشاشة (`screen`) — من إسنادات الحقل
- وضع الموثوقية (`mode`) — المشتق: واحد/متعدد
- بحث نصّي على الاسم والرقم المرجعي

## 12. ثوابت يجب ألّا تُخالَف (Invariants)

1. الحقل يُعرَّف مرة واحدة؛ لا تكرار لمعلومة واحدة في حقلين.
2. الإلزام والموثوقية وطريقة الاستخدام خصائص للإسناد، **لا للحقل**.
3. الموثوقية تُشتق من أدوار **الإدخال** فقط؛ «عرض فقط» لا يُنتج موثوقية.
4. `required` و`final` بلا معنى عند `mode === "view"`.
5. الشاشات في الإسناد **فرعية من الدور** المختار.
6. وضع الموثوقية `واحد` ⟹ القيمة معتمدة دائماً.
7. الحقل المسجّل في قاعدة البيانات لا يُحذف — يُؤرشَف، ورقمه المرجعي يبقى.
8. «النوع» نوع بيانات فقط؛ ليس وضع موثوقية.
9. الوسوم تصنيف فقط؛ لا تحدد ظهور الحقل (ذلك للإسناد).
