## ١. الخطوط المعتمدة

| النطاق | الخط | المصدر |
|---|---|---|
| العربي | **IBM Plex Sans Arabic** | Google Fonts |
| اللاتيني والأرقام | **IBM Plex Sans** | Google Fonts |

**سبب الاختيار:** الخطّان من نفس العائلة ونفس فريق التصميم، بمقاييس متطابقة (ارتفاع السين، سُمك الحدّ، شكل النهايات). النظام يعرض محتوى مختلطاً باستمرار — أرقام مراجع مثل `EJ-2026-0412`، أسماء بنوك مثل `SNB Capital`، مصطلحات مالية مثل `DSCR` — والتطابق المصمَّم يمنع التنافر البصري في هذه المواضع.

## ٢. الاستدعاء

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=IBM+Plex+Sans+Arabic:wght@400;500&family=IBM+Plex+Sans:wght@400;500&display=swap" rel="stylesheet">
```

**اشتراطات:**

- `display=swap` إلزامي — يمنع الوميض الأبيض أثناء التحميل
- وزنان فقط لكل خط: `400` و `500`. لا تستورد أوزاناً إضافية
- في بيئة الإنتاج: استضف الملفات محلياً (`woff2`) بدل Google Fonts، لسببين — الاعتماد على خدمة خارجية، ونظام النسخ الاحتياطي/العزل الشبكي لدى بعض العملاء البنكيين
- استورد `arabic` و `latin` subsets فقط. لا تستورد `latin-ext` أو `cyrillic`

## ٣. المتغيرات الأساسية

```css
:root {
  --font-ar: 'IBM Plex Sans Arabic', 'IBM Plex Sans', sans-serif;
  --font-lat: 'IBM Plex Sans', sans-serif;

  --base-size: 14px;
  --lh: 1.75;
  --sec: 12px;
  --cell-pad: 10px 12px;
}

[data-density="comfortable"] {
  --base-size: 15px;
  --sec: 13px;
  --cell-pad: 12px 14px;
}

body {
  font-family: var(--font-ar);
  font-size: var(--base-size);
  line-height: var(--lh);
  font-weight: 400;
}
```

**اشتراط:** كل مقاس خط في النظام يجب أن يشتق من `--base-size` أو من الجدول في القسم ٤. **ممنوع** كتابة `font-size: 13px` مباشرة في مكوّن.

## ٤. سُلّم المقاسات

| العنصر | المقاس | الوزن |
|---|---|---|
| عناوين الصفحات (h2) | 20px | 500 |
| عناوين الأقسام (h3) | 17px | 500 |
| النص الأساسي | 14px | 400 |
| الجداول والقوائم | 14px | 400 |
| التسميات والحقول | 13px | 400 |
| نصوص ثانوية / تلميحات | 12px | 400 |
| الأرقام في البطاقات الإحصائية | 24px | 500 |

**الحد الأدنى المطلق: 12px.** الحروف العربية تعتمد على النقاط والهمزات، وتحتها تختفي التفاصيل. لا استثناءات.

---

## ٥. قواعد ملزمة

### ٥.١ ارتفاع السطر

`line-height: 1.75` للنص العربي. **لا تنزل عن 1.6** في أي حال — الحروف المتصلة تحتاج تنفّساً رأسياً أكثر من اللاتينية.

استثناء وحيد: الأرقام الكبيرة في البطاقات الإحصائية → `line-height: 1.3`.

### ٥.٢ الأوزان

وزنان فقط: **400** للنص، **500** للتوكيد والعناوين.

**ممنوع** استخدام 600 أو 700 — يطمس عيون الحروف (ص، ض، ط، ظ) في المقاسات الصغيرة.

للتوكيد داخل نص 14px: استخدم **التباين اللوني** (`--text-primary` مقابل `--text-secondary`) لا الوزن. أنظف بصرياً وأوضح في التسلسل.

### ٥.٣ الأرقام في الجداول

```css
.num { font-variant-numeric: tabular-nums; }
```

إلزامي على كل عمود يحتوي أرقاماً (القيم، المراجع، التواريخ). بدونه لا تتحاذى الفواصل رأسياً ويصعب المسح البصري للأعمدة.

### ٥.٤ عرض السطر

```css
p { max-width: 68ch; }
```

أقصى عرض للفقرة ~68 حرفاً. الفريق يعمل على شاشات 27 بوصة — النص الممتد عبر الشاشة كاملة يُفقد القارئ موضعه عند القفز للسطر التالي.

### ٥.٥ ممنوعات تقنية

```css
/* ❌ ممنوع — يكسر ربط الحروف العربية */
text-rendering: optimizeSpeed;
font-feature-settings: "liga" 0;
letter-spacing: 0.5px;   /* يفصل الحروف المتصلة */
text-transform: uppercase; /* بلا معنى بالعربي، ويضرّ اللاتيني المختلط */
```

اترك `font-feature-settings` و `text-rendering` على الافتراضي. `letter-spacing` مسموح على اللاتيني المنفصل فقط (مثل رموز المراجع)، وبقيمة لا تتجاوز `0.3px`.

### ٥.٦ الكثافة

خيار `compact` (افتراضي) و `comfortable` متاح للمستخدم من الإعدادات.

**اشتراط:** يُخزَّن الاختيار في **ملف تعريف المستخدم في قاعدة البيانات**، لا في `localStorage`. الفريق ينتقل بين الأجهزة (مكتب، اجتماع، زيارة عميل) والإعداد يجب أن يتبعه.

## ٦. المحتوى المختلط

النظام يعرض عربي ولاتيني في نفس السطر باستمرار. القاعدة:

```html
<p>تقييم أرض بمرجع <span class="lat">EJ-2026-0412</span> لصالح <span class="lat">SNB Capital</span></p>
```

```css
.lat { font-family: var(--font-lat); }
```

**اشتراطات الاتجاه:**

- `dir="rtl"` على الحاوية الأساسية
- المحتوى اللاتيني داخل نص عربي: لا يحتاج `dir="ltr"` منفصلاً في الحالات العادية — خوارزمية BiDi تتكفّل به
- **استثناء:** رموز المراجع التي تحتوي شرطات وأرقام (`EJ-2026-0412`) قد تنعكس. إن ظهر ترتيب خاطئ، غلّفها بـ `<bdi>` أو `dir="ltr"` مع `display:inline-block`
- **اختبر دائماً:** رقم مرجع في نهاية جملة عربية متبوع بنقطة — أشهر موضع للانعكاس

---

## ٧. قائمة فحص قبل الدمج

- [ ] لا يوجد `font-size` مكتوب مباشرة خارج ملف المتغيرات
- [ ] لا يوجد وزن 600 أو 700 في أي مكان
- [ ] لا يوجد نص تحت 12px
- [ ] كل عمود أرقام عليه `tabular-nums`
- [ ] `line-height` لا يقل عن 1.6 في أي نص عربي
- [ ] الفقرات محدودة بـ `max-width`
- [ ] رموز المراجع تُعرض بترتيب صحيح داخل الجمل العربية
- [ ] الواجهة مُختبرة على `compact` و `comfortable`
- [ ] الخطوط تُحمّل بـ `display: swap`
- [ ] لا `letter-spacing` على نص عربي

## ٨. المرجع السريع

```css
:root {
  --font-ar: 'IBM Plex Sans Arabic', 'IBM Plex Sans', sans-serif;
  --font-lat: 'IBM Plex Sans', sans-serif;
  --base-size: 14px;
  --lh: 1.75;
  --sec: 12px;
  --cell-pad: 10px 12px;
}

[data-density="comfortable"] {
  --base-size: 15px;
  --sec: 13px;
  --cell-pad: 12px 14px;
}

body {
  font-family: var(--font-ar);
  font-size: var(--base-size);
  line-height: var(--lh);
  font-weight: 400;
}

h2 { font-size: 20px; font-weight: 500; }
h3 { font-size: 17px; font-weight: 500; }

.lat { font-family: var(--font-lat); }
.num { font-variant-numeric: tabular-nums; }
.sec { font-size: var(--sec); color: var(--text-secondary); }

td, th { padding: var(--cell-pad); text-align: right; }
p { max-width: 68ch; }
```
