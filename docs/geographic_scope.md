# النطاق الجغرافي للأدوار
**إجادة للتقييم العقاري — توثيق متطلبات النظام**
**النسخة:** 1.0 | **التاريخ:** 2026-06-08

---

## 1. المفهوم العام

بعض الأدوار في النظام مقيّدة بنطاق جغرافي محدد، بمعنى أن المستخدم يُسمح له بالعمل على الحالات الواقعة في مدن بعينها فقط. المدينة هي وحدة النطاق الأساسية، وداخل كل مدينة تُعرَّف المحاكم التابعة لها.

---

## 2. البنية الهرمية للنطاق

```
المنطقة الإدارية
    └── المدينة              ← وحدة النطاق الأساسية لجميع الأدوار
            └── المحاكم      ← وحدة النطاق الفرعية للمراجع الحكومي
```

---

## 3. الأدوار ذات النطاق الجغرافي

| الدور | وحدة النطاق | آلية التحديد |
|---|---|---|
| المعاين | مدينة أو أكثر | يُعيَّن عند إنشاء حساب المستخدم |
| المكتب الهندسي | مدينة أو أكثر | يُعيَّن عند إنشاء حساب المستخدم |
| المراجع الحكومي | مدينة → محاكم داخلها | يُعيَّن عند إنشاء حساب المستخدم |

> **ملاحظة:** يمكن تعيين مدينتين متقاربتين أو أكثر لنفس المستخدم عند الحاجة.

---

## 4. مصدر بيانات المحكمة في الحالة

المحكمة **ليست حقلاً يُملأ يدوياً**، بل تأتي من مسار محدد:

```
إنشاء الحالة
    ↓
تحديد المدينة  ←  من عنوان العقار
    ↓
الاستعلام في البورصة العقارية  ←  يُعيد بيانات الصك
    ↓
استخراج المحكمة من بيانات الصك  ←  تُملأ آلياً في الحالة
```

**النتيجة:** معرفة المدينة تكفي لمعرفة المحكمة، لأن المحكمة مشتقة من الصك الذي يُستعلم بعد تحديد المدينة.

---

## 5. منطق التعيين

### 5.1 السلوك الافتراضي

عند تعيين دور لحالة معينة، يعرض النظام **المؤهلين افتراضياً** بناءً على:

- **المعاين والمكتب الهندسي:** مدينة العقار ضمن نطاق المستخدم
- **المراجع الحكومي:** مدينة العقار ومحكمتها ضمن نطاق المستخدم

### 5.2 التجاوز اليدوي (خارج النطاق)

التعيين خارج النطاق **مسموح به**، مع الاشتراطات التالية:

- تتوفر في الواجهة خيار "توسيع النطاق" أو ما يعادله
- يُسجَّل التجاوز في سجل الحالة مع:
  - اسم المستخدم الذي أجرى التجاوز
  - الدور المُعيَّن خارج النطاق
  - سبب التجاوز (حقل نصي اختياري أو قائمة أسباب)
  - التاريخ والوقت

## 6. متطلبات البيانات المرجعية

### 6.1 جدول المدن — `cities`

| الحقل | النوع | الوصف |
|---|---|---|
| `city_id` | UUID | معرف فريد |
| `city_name` | string | اسم المدينة |
| `region_id` | FK | المنطقة الإدارية التابعة لها |
| `is_active` | boolean | هل المدينة نشطة في النظام |

### 6.2 جدول المحاكم — `courts`

| الحقل | النوع | الوصف |
|---|---|---|
| `court_id` | UUID | معرف فريد |
| `court_name` | string | اسم المحكمة |
| `city_id` | FK | المدينة التابعة لها |
| `is_active` | boolean | هل المحكمة نشطة |

### 6.3 نطاق المستخدم — `user_geographic_scope`

| الحقل | النوع | الوصف |
|---|---|---|
| `scope_id` | UUID | معرف فريد |
| `user_id` | FK | المستخدم |
| `city_id` | FK | المدينة المُخصصة |
| `assigned_at` | datetime | تاريخ التعيين |
| `assigned_by` | FK | من قام بالتعيين |

> المحاكم المُتاحة للمراجع الحكومي تُستنتج تلقائياً من المدن المُعيَّنة له — لا تُخزَّن محاكم بشكل منفصل في نطاق المستخدم.

### 6.4 سجل التجاوزات — `scope_override_log`

| الحقل | النوع | الوصف |
|---|---|---|
| `log_id` | UUID | معرف فريد |
| `case_id` | FK | الحالة |
| `user_id` | FK | المستخدم المُعيَّن خارج نطاقه |
| `role` | enum | الدور (معاين / مكتب هندسي / مراجع حكومي) |
| `override_by` | FK | من أجرى التجاوز |
| `reason` | text | سبب التجاوز |
| `created_at` | datetime | التاريخ والوقت |

## 7. متطلبات الواجهة

### 7.1 شاشة التعيين

- تعرض افتراضياً قائمة المؤهلين ضمن نطاق الحالة فقط
- تُظهر عدد المؤهلين المتاحين بجانب اسم كل دور
- زر أو رابط "عرض خارج النطاق" يوسّع القائمة ويضيف مؤشراً مرئياً للمستخدمين خارج النطاق
- عند اختيار مستخدم خارج النطاق تظهر رسالة تأكيد مع حقل السبب

### 7.2 إنشاء/تعديل حساب المستخدم

- قسم مخصص لتعيين المدن المسموح بها
- يُتيح اختيار مدن متعددة
- في حالة المراجع الحكومي: تُعرض المحاكم التابعة للمدن المختارة للاطلاع فقط (لا تعيين يدوي للمحاكم)

---

## 8. القواعد والقيود

1. كل مستخدم ذو نطاق جغرافي يجب أن يمتلك مدينة واحدة على الأقل في نطاقه
2. المحاكم تتبع المدن، ولا تُعيَّن باستقلالية
3. تغيير المدينة المُعيَّنة لمستخدم لا يؤثر على الحالات المُغلقة السابقة
4. التجاوز مسموح لأي مستخدم يملك صلاحية التعيين، ويُسجَّل دائماً
5. حقل المحكمة في الحالة يُملأ آلياً من الاستعلام في البورصة العقارية ولا يقبل إدخالاً يدوياً
